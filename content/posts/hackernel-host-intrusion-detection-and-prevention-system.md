## 引言

本文介绍近期设计实现的主机入侵检测与防御系统. 行文没有章法, 读起来就图一乐.

## 初衷

开源社区有类似的产品, 实现方案也大体一致, 如果仅仅是为了达成某项需求, 没必要再造轮子. 我的初衷是学习内核相关的技术, 而个人认为没有什么方法比直接做一个出来更能验证学习成果.

[最初的提交](https://github.com/lanthora/hackernel/commit/3fbf8393aba2423503822741e22873cf50f5bf0e)非常简陋, 这大概是每个学习内核的人, 动手写的第一段代码, 在内核模块加载和移除时打印日志. 如果你对内核感兴趣却不知道如何下手, 看过最初的提交, 好像也挺简单? 如果这段代码能驱使任何人动手写第一段内核代码, 将是我莫大的荣幸.

## 名称的由来

在最初的提交中就确定项目名为 hackernel, 这是 hack 和 kernel 的组合拼写. 
关于 hack 的定义可以参考[维基百科](https://en.wikipedia.org/wiki/Kludge#Computer_science). 
项目实现过程中采用[一些非常规的手段对内核数据进行修改](/posts/disable-kernel-memory-write-protection/). 
因此 hackernel 是个恰好符合项目的特点的名称.

## 核心功能实现原理

本项目内核模块提供出来的功能,就是核心功能,也是可完成功能的上限.

原理上: 1.通过修改系统调用表, 将系统调用修改为自定义的 hook 函数, 根据入参决定是否继续调用真实的系统调用; 2.直接使用内核提供的 hook 函数, 如网络相关的 netfilter hook 点.

hook 函数有能力决定是否继续向下层调用.允许向下调用并记录行为为检测,根据策略禁止向下调用为防御.也就契合了标题中的`检测与防御系统`.

## 第一层封装

核心功能部分提到的`策略`, 总归要来自用户态, 标题中的`入侵`也行为也由用户态定义. 
同时为了方便使用, 实现了一个用户态的进程, 让它与内核模块作为一层封装,通过 Unix Domain Socket 对外提供 [API](https://hackernel.org/api.html). 
通过 API 可以清晰的认识本项目所能提供的功能.

于是本文提到的的第一个项目 [hackernel](https://github.com/lanthora/hackernel) 出现.

## 第二层封装

第一层封装相比于直接与内核模块通信, 对于开发人员已经足够友好, 但对于非技术敏感的用户, 门槛依旧难以跨越.

于是有了第二层封装: 通过 Go 把这些第一层 API 封装成函数以方便开发者调用.

敏感的朋友可能已经发现: 第二层封装本质上是作为客户端使用第一层封装提供的服务, 同时又作为服务被其他应用调用.
为了解耦和方便扩展, 选择用 Go 实现并作为一个项目独立维护. 
选择 Go 的根本原因是,当时可选的语言只有 Go 和 Rust, 而 Go 刚好有我需要的库.(如果你认为有更好的选择,欢迎自行实现第二层封装)

此时项目就需要一个名字, 当时 _海贼王_ 三大古代兵器只有天王乌拉诺斯的还没有揭晓, 于是选择它作为新项目的名字.

于是本文提到的的第二个项目 [Uranus](https://github.com/lanthora/uranus) 出现.

## 应用

显然第二层封装是库, 于是在第二层封装之上, 实现了[三个应用](https://github.com/lanthora/uranus/tree/master/cmd). 
这三个应用会在编译 Uranus 时编译出来, 也是提供给最终用户的产品.

### uranus-sample

示例程序, 展示库的使用方法, 验证各底层模块是否正常工作.

### uranus-telegram

目前对这套程序推荐的使用方式, 也是作者自己使用的方式. 
通过 Telegram 机器人, 上报系统中正在发生的进程创建事件. 
监控入侵事件.

### uranus-web

以网页的形式展示功能.目前后端工作已经基本完成. 前端尚不可用.
本文提到的第三个项目, 前端项目 [venus](https://github.com/lanthora/venus) 目前处于停滞状态.
在有人提出明确的需求且愿意主动维护这个项目前, 将长期处于停滞状态.

在选择前端项目名称时, 发现 uranus 是罗马神话的神, 考虑到这是个前端项目, 罗马神话里维纳斯可能是最合适的选择.

## 安装

这里你将看到本文提到的第四个项目,即罗马神话中的创造之母 [tellus](https://github.com/lanthora/tellus).
这个项目包含前三个项目.并提供编译安装方法.

对于 Arch Linux 用户,提供 AUR 方式安装.使用 yay 或者任何你喜欢的 AUR Helper.

```bash
yay -S tellus-git
```
